# PRD 產品需求文件：食譜收藏系統

## 1. 專案概述

### 背景與動機
現代人常在各個平台（網站、社群媒體、書籍）看到喜歡的食譜，但缺乏一個統整的系統來記錄與管理。為了解決這個問題，我們需要一個專門的「食譜收藏系統」，讓使用者可以方便地儲存、瀏覽並搜尋自己感興趣的食譜。

### 目標用戶
- **一般人（一般使用者）**：想要尋找靈感、儲存個人喜愛食譜並記錄料理筆記的使用者。
- **管理員**：負責維護系統運作、管理全站公開食譜內容與使用者帳號的系統管理者。

### 核心價值主張
提供一個簡單、直覺的平台，讓使用者能輕鬆收藏並管理喜愛的食譜，包含詳細的所需食材與作法介紹，提升日常料理的便利性與樂趣。

---

## 2. 功能需求

### 主要功能與使用者故事

1. **儲存食譜**
   - 作為 **一般人**，我希望 **能夠將感興趣的食譜儲存到我的個人收藏夾中**，以便 **日後可以快速找到並參考**。
   - 作為 **一般人**，我希望 **能夠自己新增食譜資訊**，以便 **記錄自己的私房菜單**。

2. **瀏覽食譜**
   - 作為 **一般人**，我希望 **能夠在一個直覺的介面中瀏覽所有食譜或我的收藏清單**，以便 **尋找今天的料理靈感**。

3. **搜尋食譜**
   - 作為 **一般人**，我希望 **能夠透過關鍵字或分類來搜尋食譜**，以便 **快速找到特定的料理**。

4. **所需食材介紹**
   - 作為 **一般人**，我希望 **每份食譜都能清楚列出所需的食材清單與份量**，以便 **我可以方便地準備採買**。

5. **作法介紹**
   - 作為 **一般人**，我希望 **每份食譜都有詳細的步驟式作法介紹**，以便 **我能跟著步驟順利完成料理**。

6. **系統管理（管理員專屬）**
   - 作為 **管理員**，我希望 **能夠編輯或刪除平台上的公開食譜內容**，以便 **維持平台內容的品質與正確性**。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Flask (Python)。
- **前端渲染**：使用 Jinja2 模板引擎搭配 HTML/CSS/JavaScript。
- **資料庫**：使用 SQLite 進行資料儲存，符合輕量與易部署的需求。

### 效能與安全考量
- **效能**：網站回應時間應保持在合理範圍（如頁面載入時間低於 2 秒）。圖片若有上傳功能，需進行壓縮與大小限制。
- **安全性**：
  - 防止 SQL Injection 攻擊（應使用 ORM 或 Prepared Statements）。
  - 防止 XSS 攻擊（Jinja2 預設開啟 Autoescaping，需確保輸入欄位有適當過濾）。
  - 密碼需經過雜湊（如 bcrypt）處理後再存入資料庫。
  - 實作基本的身分驗證與授權機制（Session-based authentication）。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備)
- 使用者註冊與登入（區分一般使用者與管理員）。
- 瀏覽公開食譜列表與詳細頁面。
- 新增、編輯、刪除食譜（包含名稱、所需食材、作法介紹）。
- 收藏/取消收藏食譜功能，並能查看「我的收藏」清單。
- 基礎的關鍵字搜尋功能（針對食譜名稱或食材）。

### Should Have (應該具備)
- 食譜分類或標籤功能（如：主食、甜點、湯品）。
- 使用者個人資料編輯。
- 簡易的管理員後台（可管理所有食譜與使用者）。

### Nice to Have (有了更好)
- 食譜圖片上傳與展示。
- 對食譜發表評論或評分功能。
- 一鍵產生食材採買清單。
